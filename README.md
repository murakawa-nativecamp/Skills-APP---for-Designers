# Skills-APP---for-Designers

NativeCamp デザイナー向けの Claude Cowork スキル集です。
このリポジトリは **Cowork プラグインのマーケットプレイス** になっています。
GitHub に詳しくなくても、下の手順どおりに進めれば使えます。

---

## デザイナー向け：使い方

### ① 最初に1回だけ：マーケットプレイスを登録してインストール

**Cowork（デスクトップアプリ）の場合**

1. 右上の **Customize** →  **Plugins** を開く
2. 「Personal plugins」の **＋** ボタンを押す
3. **Add marketplace** を選ぶ
4. 次の文字列をそのまま貼り付ける：
   ```
   murakawa-nativecamp/Skills-APP---for-Designers
   ```
5. **Add** を押す
6. 一覧に出てくる **nativecamp-design-skills** の **Install** を押す
7. **Done** を押す

これで完了です。使うときは **`/コマンド` として送る必要はありません**。依頼文を普通に書くだけで、内容に応じてスキルが自動で発動します（→ 後述「補足（確実に発動させたいとき）」）。

### ② 更新を取り込みたいとき

新しいバージョンが公開されたら、次のコマンドを実行すると最新になります：

```
/plugin marketplace update murakawa-nativecamp/Skills-APP---for-Designers
```

（自動更新がオンの場合は、起動時に自動で最新化されます。）

## プロンプト例（デザイナー向け）

> ⚠️ **その前に：先にスキル（プラグイン）の設定を忘れずに。**
> プロンプトを使う前に、上の **「① 最初に1回だけ：マーケットプレイスを登録してインストール」** を済ませて、プラグイン **nativecamp-design-skills** がインストール済みになっていることを確認してください。
> 会話の入力欄で `/` を押したときに **nativecamp-figma-design-fix** と **figma-to-claude-design-prototype** が一覧に出れば、インストールは完了しています。
> ただし **`/スキル名` をコマンドとして送る必要はありません**（送ると「Unknown command」エラーになります）。下の例文のように、普通に依頼文を書くだけでOKです。

## ① nativecamp-figma-design-fix（DSで画面を組み直す）

このスキルは「**参考フレーム**（お手本にしたい画面）」と「**作成フレーム**（実際に組み直す、既存の空きフレームなど）」の2つの Figma URL を渡して使います。

事前準備として、Figma 上に**作成先となるフレームを用意**しておき、そのフレームを選択した状態で URL をコピーしてください（URL末尾の `?node-id=...` がそのフレームを指します）。

### 基本の頼み方

```
この参考フレームに合わせて、作成フレームをDesign Systemで組み直してください。
テキストは参考と同じ、レイアウト・色・余白・角丸・線幅は全部DS変数にバインドで。

参考フレーム（お手本）:
https://www.figma.com/design/HKEeQO4ltPu3moj0U0FCwg/App-Master-File-for-student?node-id=1-491

作成フレーム（ここに組む）:
https://www.figma.com/design/HKEeQO4ltPu3moj0U0FCwg/App-Master-File-for-student?node-id=◯◯-◯◯
```

> `node-id` の部分は、それぞれのフレームを選んでコピーした URL に置き換えてください。
> 参考フレームと作成フレームは同じファイルでも、別ファイルでも構いません。

### バリエーション例

- **タブバーの扱いを指定**
  ```
  作成フレームを参考フレームに合わせて作って。タブバーは参考に無いので削除して。
  参考: <参考フレームのURL>
  作成: <作成フレームのURL>
  ```

- **複数画面をまとめて**
  ```
  下の参考に合わせて、作成フレーム①②③を順番に組み直して。テキストは参考どおり。
  参考: <参考フレームのURL>
  作成①: <フレーム1のURL>
  作成②: <フレーム2のURL>
  作成③: <フレーム3のURL>
  ```

- **ボタンや色だけ直したい**
  ```
  この画面のボタンを Primary / Secondary に整えて、色・余白・角丸・線幅を全部DS変数にバインドして。
  対象フレーム: <作成フレームのURL>
  ```

> 既存の似た画面がある場合は、スキルが自動で「App Master File for student」を参照して作り方を踏襲します。

## ② figma-to-claude-design-prototype（動くプロトタイプにする）

Figmaで作った画面たちを、**Claude Design 上で実際にタップ・入力して試せる「動くプロトタイプ」**に変換するスキルです。エラー表示の出方、フォーカス時の見た目、画面遷移の流れを操作しながら確認できます。

事前準備は2つだけです：

- Figma の対象ページ（またはフレーム）の URL をコピーしておく（URL末尾の `?node-id=...` 付き）
- ブラウザで claude.ai（Claude Design）に**ログインしておく**

### 基本の頼み方

```
この Figma の画面たちをプロトタイプ化して、動作確認できるようにしてください。

対象: https://www.figma.com/design/◯◯◯/◯◯◯?node-id=◯◯-◯◯
```

### バリエーション例

- **エラー表示の確認がしたい**
  ```
  この決済フローのエラー表示（未入力・形式エラー）を Claude Design で試せるようにして。
  対象: <FigmaのURL>
  ```

- **画面遷移とフォーカスの確認がしたい**
  ```
  この画面たちを触れるようにして。フォーカス時の見た目と画面遷移を確認したい。
  対象: <FigmaのURL>
  ```

> 💡 Figma 側のフレーム名が `Login_Default` / `Login_Error_Email` のような「画面名＋状態」の形になっていると、同じ画面の状態違いとして自動でまとめて、操作で切り替わるプロトタイプになります。

## 補足（確実に発動させたいとき）

自動発動は依頼文の表現に依存するため、「うまく発動しない」ケースもあり得ます。その保険として、デザイナーには以下を伝えておくと安心です。

- 普段は**自然文でOK**（コマンド不要）。
- 発動しないときは、依頼の頭に **「nativecamp-figma-design-fix を使って」「figma-to-claude-design-prototype を使って」** のようにスキル名を一言添える。

まとめると、**コマンドの指示は基本不要**です。プラグインを配布・有効化してもらい、「どういう言い方をすれば発動するか（"参考に合わせて" ／ "Figmaを編集して" ／ "プロトタイプ化して" ／ "エラー表示を試したい" 等）」の例文だけ共有すれば、他のデザイナーもすぐ使えます。

## メンテナー向け（編集する人）

- スキルの本体は `plugins/nativecamp-design-skills/skills/<スキル名>/SKILL.md` です。
- このフォルダがGitHubと自動同期されています。`SKILL.md` を編集して保存すれば、
  定期的に自動で GitHub に push され、バージョンも自動で上がります。
- 新しいスキルを追加するときは、`skills/` の下に `<スキル名>/SKILL.md` を作ります。
- 編集役は1人に集約するのが安全です（デザイナーはGitHubを直接触らない運用）。
