# Skills-APP---for-Designers

NativeCamp デザイナー向けの Claude Cowork スキル集です。
このリポジトリは **Cowork プラグインのマーケットプレイス** になっています。
GitHub に詳しくなくても、下の手順どおりに進めれば使えます。

---

## デザイナー向け：使い方

### ① 最初に1回だけ：マーケットプレイスを登録してインストール

**Cowork（デスクトップアプリ）の場合**

1. 右上の **Customize** →  **Plugins** を開く
2. 「Personal plugins」の **＋** ボタンを押す
3. **Add marketplace** を選ぶ
4. 次の文字列をそのまま貼り付ける：
   ```
   murakawa-nativecamp/Skills-APP---for-Designers
   ```
5. **Add** を押す
6. 一覧に出てくる **nativecamp-design-skills** の **Install** を押す
7. **Done** を押す

これで完了です。使うときは **`/コマンド` として送る必要はありません**。依頼文を普通に書くだけで、内容に応じてスキルが自動で発動します（→ 後述「補足（確実に発動させたいとき）」）。

### ② 更新を取り込みたいとき

新しいバージョンが公開されたら、次のコマンドを実行すると最新になります：

```
/plugin marketplace update murakawa-nativecamp/Skills-APP---for-Designers
```

（自動更新がオンの場合は、起動時に自動で最新化されます。）

## ① nativecamp-figma-design-fix（Figma画面をDSで作る／直す）

このスキルには **2つの使い方** があります。どちらも「**作成フレーム**（実際に組む先。既存の空きフレームなど）」の Figma URL が必要です（Figma上に作成先フレームを用意し、選択した状態で URL をコピー＝URL末尾の `?node-id=...` がそのフレームを指します）。

- **パターンA：参考あり（お手本に合わせて組み直す）** … 参考フレームのテキスト・構成に合わせ、レイアウトを Design System できれいに整える。
- **パターンB：参考なし（指示から新規に作る）** … 参考フレームが無くても、依頼の指示だけで DS に沿って画面を作る。既存の「App Master File for student」の似た画面パターンを自動で踏襲します。

### 基本の頼み方（A：参考あり）

```
この参考フレームに合わせて、作成フレームをDesign Systemで組み直してください。
テキストは参考と同じ、レイアウト・色・余白・角丸・線幅は全部DS変数にバインドで。

参考フレーム（お手本）:
https://www.figma.com/design/HKEeQO4ltPu3moj0U0FCwg/App-Master-File-for-student?node-id=1-491

作成フレーム（ここに組む）:
https://www.figma.com/design/HKEeQO4ltPu3moj0U0FCwg/App-Master-File-for-student?node-id=◯◯-◯◯
```

> `node-id` の部分は、それぞれのフレームを選んでコピーした URL に置き換えてください。
> 参考フレームと作成フレームは同じファイルでも、別ファイルでも構いません。

### 基本の頼み方（B：参考なし＝指示だけで作る）

参考フレームは渡さず、**作成フレームのURL＋作ってほしい画面の説明**を渡します。画面の要素（見出し・本文・選択肢・ボタン等）を箇条書きで伝えるほど、意図どおりに組み上がります（文言まで指定すればそのまま反映）。

```
この作成フレームに、退会理由を選ぶ画面を Design System で作ってください。
・見出し「退会の理由を教えてください」＋説明文
・選択肢を4つ（単一選択）
・下部に Primary ボタン「次へ」
テキスト以外（レイアウト・色・余白・角丸・線幅）は全部DS変数にバインドで。
似た既存画面があれば App Master File for student に合わせて。

作成フレーム（ここに組む）:
https://www.figma.com/design/HKEeQO4ltPu3moj0U0FCwg/App-Master-File-for-student?node-id=◯◯-◯◯
```

> 参考が無い場合でも、DS（ダークモード）に沿ってゼロから組みます。方向性だけ（例：「〜な画面を作って」）でも作れますが、要素を箇条書きにすると精度が上がります。

### バリエーション例

- **タブバーの扱いを指定**
  ```
  作成フレームを参考フレームに合わせて作って。タブバーは参考に無いので削除して。
  参考: <参考フレームのURL>
  作成: <作成フレームのURL>
  ```

- **複数画面をまとめて**
  ```
  下の参考に合わせて、作成フレーム①②③を順番に組み直して。テキストは参考どおり。
  参考: <参考フレームのURL>
  作成①: <フレーム1のURL>
  作成②: <フレーム2のURL>
  作成③: <フレーム3のURL>
  ```

- **ボタンや色だけ直したい**
  ```
  この画面のボタンを Primary / Secondary に整えて、色・余白・角丸・線幅を全部DS変数にバインドして。
  対象フレーム: <作成フレームのURL>
  ```

> 既存の似た画面がある場合は、スキルが自動で「App Master File for student」を参照して作り方を踏襲します。

## ② nativecamp-app-design-storyboard（デザインをHTML化して開発者に渡す）

Figmaのアプリ画面デザインを、**本番相当に忠実なHTML（画像・アイコン・キャラのLottieまで埋め込んだ自己完結ファイル）＝「画面ストーリーボード」**に再現するスキルです。ブラウザで開くだけで確認でき、最終的に**開発者へ実装の下地としてzipで渡せます**。※これは本番実装そのものではなく、忠実な参照コードです。**アプリ専用（PC/Webは別デザインルールのため対象外）**。

事前準備：

- Figma の対象フレーム/グループの URL をコピー（URL末尾の `?node-id=...` 付き）
- 画面数が多い場合はグループ（例：Variant A / B）单位で渡すと一括で組めます

### 基本の頼み方

```
この Figma のアプリ画面デザインをHTML化して、画面ストーリーボードにしてください。
実アセット（アイコンはSVG、画像はWEBP、キャラはLottie）を使って本番相当に忠実に。

対象: https://www.figma.com/design/◯◯◯/◯◯◯?node-id=◯◯-◯◯
```

### 進め方（zipは最後）

1. 公開されたアーティファクト（HTML）の**URLをブラウザで目視確認**
2. 気になる箇所を画面単位で指摘 → 修正 → 同じURLに再公開（何度でも）
3. OKになったら **「開発者に渡すzipにして」** で、HTML＋実アセット＋READMEをzip化

### バリエーション例

- **A/B 両方作る**
  ```
  この登録フローの Variant A と B を両方HTML化して、それぞれアーティファクトにして。
  A: <FigmaのURL>   B: <FigmaのURL>
  ```

- **修正だけ頼む**
  ```
  ストーリーボードの9番、カードのタイムラインの線がはみ出てるので dot 間だけに直して。
  ```

- **納品用にまとめる**
  ```
  レビューOKなので、AとBのHTMLをまとめて開発者に渡すzipにしてください。
  ```

> 💡 写真などの画像は「角丸をかける前の元素材（フィル原本）」をWEBPで書き出し、角丸はコード側で付けます（開発者がそのまま流用しやすいように）。

## 補足（確実に発動させたいとき）

自動発動は依頼文の表現に依存するため、「うまく発動しない」ケースもあり得ます。その保険として、デザイナーには以下を伝えておくと安心です。

- 普段は**自然文でOK**（コマンド不要）。
- 発動しないときは、依頼の頭に **「nativecamp-figma-design-fix を使って」「nativecamp-app-design-storyboard を使って」** のようにスキル名を一言添える。

まとめると、**コマンドの指示は基本不要**です。プラグインを配布・有効化してもらい、「どういう言い方をすれば発動するか（"参考に合わせて" ／ "Figmaを編集して" ／ "デザインをHTML化して" ／ "開発者に渡すzipにして" 等）」の例文だけ共有すれば、他のデザイナーもすぐ使えます。

## メンテナー向け（編集する人）

- スキルの本体は `plugins/nativecamp-design-skills/skills/<スキル名>/SKILL.md` です。
- このフォルダがGitHubと自動同期されています。`SKILL.md` を編集して保存すれば、
  定期的に自動で GitHub に push され、バージョンも自動で上がります。
- 新しいスキルを追加するときは、`skills/` の下に `<スキル名>/SKILL.md` を作ります。
- 編集役は1人に集約するのが安全です（デザイナーはGitHubを直接触らない運用）。
