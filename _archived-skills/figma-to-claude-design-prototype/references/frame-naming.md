# フレーム命名規則と状態の統合ルール

Figma 側のフレーム名は「`画面名` + `状態サフィックス`」で構成される。
同じ画面名を持つフレーム群は **1画面として実装し、操作で状態を切り替える**。

## 状態サフィックスの定義

| サフィックス | 意味 | プロトタイプでの発火条件 |
|---|---|---|
| `_Default` | 初期表示 | 画面を開いた直後 |
| `_Focus_{フィールド名}` | 該当フィールドのフォーカス状態 | そのフィールドをタップ/クリック |
| `_Filled_{バリエーション}` | 入力済み状態 | 入力完了後の表示（例: `PwHidden` / `PwVisible` はパスワード表示トグルの OFF / ON） |
| `_Empty` | 未入力のまま送信した際のエラー | 空のまま送信ボタンを押す |
| `_Error_{フィールド名}` | 該当フィールドの形式・要件エラー | 不正な値を入力して送信（形式不正・桁数不足など） |

## 統合ルール ※重要

- 上記サフィックスを持つフレーム群は、**別画面にせず1画面に統合**する。
  状態の見た目の差分（枠線色・エラーメッセージ・アイコン等）を、
  フォーカス・入力・送信などの**操作によって切り替える**実装にする。
- 例: `Login_Default` / `Login_Focus_Email` / `Login_Filled_PwVisible` /
  `Login_Empty` / `Login_Error_Email` → 実装は **Login 1画面**。

## オーバーレイ扱いのフレーム

- **Action Sheet 系**（`ActionSheet_◯◯`、`_Sheet_◯◯` など）と
  **`block_sheet` 系**のフレームは、独立した画面ではなく
  **親画面の上に重なるオーバーレイ**として実装する。
- 背景は親画面が透けて見える半透明スクリム。閉じる操作（キャンセル/スクリムタップ）で
  親画面に戻る。

## マッピング表の作り方

1. `get_metadata` で取得した全フレーム名を、サフィックスを切り落とした「画面名」でグルーピング
2. 画面ごとに状態フレームを列挙し、発火条件を書き添える
3. オーバーレイは「どの画面から開くか」を明記する
4. どのサフィックスにも当てはまらない名前のフレームは、ユーザーに役割を確認する
