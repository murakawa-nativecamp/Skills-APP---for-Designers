# プロトタイプ検証チェックリスト

生成されたプロトタイプを**実際に操作して**確認する。各周回でこのリストを上から順に実施する。

## 1. 画面遷移
- [ ] 全画面への遷移が機能する（マッピング表の全画面に到達できる）
- [ ] 戻る操作が機能し、入力途中の画面では離脱確認シートが出る
- [ ] アクションシートが正しい画面から開き、閉じると元の画面に戻る

## 2. 状態切り替え（各画面）
- [ ] 未入力で送信 → _Empty のエラーが正しい見た目で表示される
- [ ] 形式不正の値で送信 → _Error_{フィールド名} が正しい見た目で表示される
- [ ] 要件エラー（桁数・文字種など）も同様に表示される
- [ ] エラー後に正しい値を再入力すると、エラー表示が解消される
- [ ] フィールドタップで _Focus_ の見た目になる

## 3. 入力ふるまい
- [ ] パスワード表示トグルが動く（伏字 ⇔ 平文）
- [ ] カード番号の4桁区切り自動整形が動く
- [ ] カードブランド自動判定が動く（番号先頭でアイコンが切り替わる）
- [ ] 有効期限の MM/YY 自動整形、CVC の桁数制限が動く

## 4. 見た目の比較（各画面 × 各状態）
各画面のスクリーンショットを撮り、Figma の PNG（ファイル名 = フレーム名）と並べて比較:
- [ ] 文言（一字一句・改行位置）
- [ ] 色（背景・テキスト・ボタン・エラー色）
- [ ] 余白（画面左右 padding・要素間の間隔）
- [ ] フォント（ファミリー・ウェイト・サイズ）

## 修正指示の書き方 ※画面名を必ず入れる

差分は1件ずつ、**フレーム名つき**で Claude Design に送る:

> 「`{フレーム名}` と比べて `{差分の内容}` が違う。`{正しい値}` にして」

例:
> 「`Payment_Error_CardNumber` と比べて、エラーメッセージの文言が違う。
> 『カード番号をご確認ください』にして」

> 「`Login_Default` と比べて、ボタンの背景色が違う。`#F1890E` にして」
