## 共通インタラクション要件
- フォーカス: フィールドをタップすると対応する _Focus_ 状態の見た目になる
- バリデーション: 送信時に実行。未入力なら _Empty、形式・要件エラーなら _Error_ の
  見た目を表示する。エラー後に正しい値を再入力すると、エラー表示が解消される
- パスワード: 表示トグル（目アイコン）で伏字 ⇔ 平文を切り替え
  （_Filled_PwHidden / _Filled_PwVisible に対応）
- カード番号: 入力中に4桁区切りで自動整形し、番号からブランド（Visa/Mastercard/JCB/Amex 等）を
  自動判定してブランドアイコンを切り替える
- 有効期限: MM/YY 形式に自動整形
- CVC: 桁数制限（Amex は4桁、その他は3桁）
- プログレスバー: 複数ステップのフローでは現在ステップを表示し、遷移に合わせて進める
- アクションシート: 指定のフレームをオーバーレイとして表示（半透明スクリム +
  下からのシート。スクリムタップ/キャンセルで閉じる）
- 戻る操作: 入力途中の画面で戻ろうとしたら離脱確認シートを表示する
- ビューポート: モバイル 375px 幅基準

## 埋め方のメモ

- `{画面と状態のマッピング表}` … SKILL.md 手順2で作った表をそのまま貼る。
  各行に「状態フレーム名（= 添付画像のファイル名）」と発火条件を含めること。
- `{色・フォント・余白の数値}` … `get_design_context` で取得した値を画面ごとに列挙。
  最低限: 背景色 / テキスト色（通常・エラー）/ ブランド色 / フォントファミリーと
  ウェイト / 主要な余白（画面左右 padding、要素間 gap）。
- 対象フローに存在しない要件（例: カード入力が無い）は該当行を削ってよい。
  それ以外の行は削らない。
